# Update Bundle Skill

## Trigger

After ANY implementation work that changes:
- Behavior
- Interfaces
- Configuration
- Dependencies

## Action

Automatically prepare updates for:

### STATUS.md
- Current project state
- What's done
- What's next
- Any blockers

### CHANGELOG.md
- What changed (user-facing description)
- Why it changed
- Version/date

### Documentation
- README updates needed
- API doc updates needed
- Or "None"

### KNOWN_ISSUES.md
- New limitations discovered
- Workarounds
- Or "None"

## Format

```markdown
## Update Bundle

### STATUS.md
[Content to add/update]

### CHANGELOG.md
[Entry to add]

### Documentation
[Updates needed or "None"]

### KNOWN_ISSUES.md
[Issues to add or "None"]
```
